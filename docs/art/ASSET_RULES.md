# ASSET_RULES

## Naming

- UI assets: `ui_<screen>_<purpose>`
- Gameplay sprites: `gp_<category>_<name>`
- Marketing images: `mk_<type>_<version>`

## Formats

- UI source: layered editable file plus exported PNG/WebP
- Gameplay sprites: PNG or texture atlas source
- Marketing: PNG, source PSD/fig equivalent tracked outside repo if needed

## Placeholder Policy

- Placeholder shapes are allowed for MVP logic validation
- Final asset replacement requires:
  - asset version
  - prompt version or human-source reference
  - QA art verdict
